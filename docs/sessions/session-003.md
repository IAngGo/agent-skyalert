## Decisions made

- **Hand-written migration instead of autogenerate** — no live PostgreSQL available
  during generation. The result is identical since we control the models. Autogenerate
  can be used for all future migrations once Docker is running.
- **`migrate` service with `service_completed_successfully`** — api/worker/beat only
  start after migrations finish. Prevents "table does not exist" errors on first boot.
- **`compare_type=True` in env.py** — makes future autogenerate migrations detect
  column type changes (e.g. String(255) → String(512)), not just new/dropped columns.
- **`NullPool` in online migration runner** — Alembic migrations run once and exit;
  connection pooling adds no value and can leave idle connections open.
- **Indexes on `searches.user_id`, `searches.is_active`, `price_history.search_id`,
  `price_history.scraped_at`, `alerts.search_id`, `alerts.status`** — these are the
  exact columns used in WHERE clauses by the repositories. Added proactively.

## Concepts covered

**[MIGRATIONS] Alembic env.py as the bridge**
- What: `env.py` imports `Base.metadata` (so Alembic sees all tables) and sets
  `sqlalchemy.url` from environment (so no credentials in alembic.ini).
- Why: Without this wiring, `alembic revision --autogenerate` produces empty migrations.
- Where: `alembic/env.py` lines 20–40.

**[MIGRATIONS] upgrade / downgrade symmetry**
- What: Every migration has an `upgrade()` and a matching `downgrade()`.
  `alembic upgrade head` applies all; `alembic downgrade -1` rolls back one.
- Why: Enables safe rollbacks in production without manual SQL.
- Where: `b0828dd2b08a_initial_schema.py` — `downgrade()` drops tables in reverse FK order.

**[DOCKER] Service dependency with health checks**
- What: `depends_on` with `condition: service_healthy` waits for a service's
  healthcheck to pass before starting the dependent service.
- Why: Without this, `api` starts before PostgreSQL is ready and crashes immediately.
- Where: `docker-compose.yml` — `migrate` waits for `db` healthy; `api/worker/beat`
  wait for `migrate` completed successfully.
